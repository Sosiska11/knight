# План по добавлению отображения количества устройств в профиле

Этот план описывает реализацию функции отображения количества активных подключений (устройств) в личном кабинете пользователя, а также исправление проблемы частых таймаутов при авторизации в панели 3x-ui.

---

## Предлагаемые изменения

### 1. Бэкенд взаимодействия с 3x-ui
#### [MODIFY] [xui-api.js](file:///c:/Users/alexs/OneDrive/Рабочий стол/code/bot/src/xui-api.js)
* Увеличить таймаут первого запроса в `login()` с 5000мс до 15000мс.
* Добавить метод `getClientIps(email)` для выполнения POST-запроса на `/panel/api/clients/ips/:email`. Метод возвращает массив уникальных активных IP-адресов клиента.

### 2. Telegram Бот
#### [MODIFY] [bot.js](file:///c:/Users/alexs/OneDrive/Рабочий стол/code/bot/src/bot.js)
* В методе `showProfile(ctx)` запрашивать список активных IP-адресов через `xuiApi.getClientIps(activeSub.client_email)`.
* Выводить в личном кабинете количество подключенных устройств:
  `• <b>Активно устройств:</b> <code>${activeIps.length} из ${activeSub.limit_ip || 1}</code>`

## План проверки

### Автоматизированные тесты
- Запустить `scratch/test-online-api.js` и убедиться, что логин проходит без ошибок и метод `getClientIps` возвращает корректный результат (пустой массив или массив с IP) с реального сервера 3x-ui.

### Ручная проверка
- Открыть личный кабинет (профиль) в боте с активной подпиской и проверить вывод количества подключенных устройств (например, `0 из 1` или `1 из 1`).
