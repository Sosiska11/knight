# Инструкция: Что делать, если перестал работать обход блокировок (LTE/Обходка)

Если у ваших пользователей перестали работать конфигурации с пометкой `LTE | Обходка`, это означает, что мобильные операторы заблокировали транзитный порт `16605` в рамках режима "белых списков" (белые списки обычно пропускают трафик только через порты `80` и `443`).

Так как текущий транзитный сервер использует NAT (общий IP-адрес) и не позволяет занимать внешние порты `80` или `443`, вам потребуется перенести транзитный узел на **выделенный VPS** с собственным публичным IP-адресом.

Ниже приведена пошаговая инструкция, как это сделать.

---

## Шаг 1. Покупка выделенного VPS в России
Вам нужен виртуальный сервер (VDS/VPS) на базе Linux (рекомендуется **Ubuntu 22.04 / 20.04**) от российского облачного провайдера, чьи IP-адреса находятся в белых списках операторов:
1. **VK Cloud** (диапазоны `95.163.x.x` — рекомендуемый выбор, самый дешевый вариант).
2. **Yandex Cloud** (диапазоны `158.160.x.x` или `84.201.x.x`).
3. **Timeweb Cloud** или любой другой провайдер с выделенным IP в РФ.

> [!IMPORTANT]
> При покупке обязательно выбирайте выделенный (публичный) IP-адрес, а не NAT.

---

## Шаг 2. Настройка файрвола в панели провайдера
После активации VPS зайдите в консоль управления провайдера (VK Cloud или Yandex Cloud) и в настройках сети/брандмауэра (Security Groups) откройте доступ:
- Входящий TCP: порт `2222` (для вашего SSH).
- Входящий TCP: порты `80` и `443` (для трафика обхода).

---

## Шаг 3. Настройка нового транзитного сервера через SSH
Подключитесь к вашему новому серверу через терминал:
```bash
ssh root@IP_НОВОГО_СЕРВЕРА
```

### 1. Перенос стандартного порта SSH
Поскольку порт `443` и `22` на сервере будут заняты мультиплексором `sslh`, нам нужно перевесить стандартную службу SSH на порт `2222`.

Откройте файл конфигурации SSH:
```bash
nano /etc/ssh/sshd_config
```
Найдите строку `#Port 22`, раскомментируйте её и измените на:
```text
Port 2222
```
Сохраните файл (`Ctrl+O`, затем `Enter`) и выйдите (`Ctrl+X`).

Перезапустите службу SSH, чтобы применить настройки:
```bash
systemctl restart sshd
```
> [!WARNING]
> Не закрывайте текущую сессию терминала, пока не проверите подключение в новом окне!
> Откройте новое окно терминала и попробуйте подключиться по новому порту: `ssh root@IP_НОВОГО_СЕРВЕРА -p 2222`.

---

### 2. Установка и настройка мультиплексора `sslh`
Установите `sslh` на сервере:
```bash
apt update && apt install -y sslh
```
При установке выберите запуск в режиме `standalone`.

Откройте файл конфигурации `sslh`:
```bash
nano /etc/default/sslh
```
Замените содержимое файла на следующее (замените `141.11.197.6` на IP вашего основного сервера VPN, если он изменился):
```text
RUN=yes
DAEMON=sslh
DAEMON_OPTS="--user sslh --listen 0.0.0.0:443 --listen 0.0.0.0:80 --ssh 127.0.0.1:2222 --tls 141.11.197.6:8443"
```
*Эта настройка заставляет сервер слушать порты 80 и 443. Если входящий запрос является SSH, он пересылается локально на порт 2222. Если запрос является TLS (VLESS Reality), он перенаправляется на ваш основной зарубежный сервер на порт 8443.*

Сохраните файл и выйдите. Перезапустите службу `sslh`:
```bash
systemctl restart sslh
systemctl enable sslh
```

Убедитесь, что `sslh` успешно занял порты `80` и `443`:
```bash
ss -tlpn | grep sslh
```
Вы должны увидеть строки с `0.0.0.0:80` и `0.0.0.0:443`.

---

## Шаг 4. Обновление настроек в Telegram-боте
Теперь вам нужно указать боту адрес нового транзитного сервера.

1. На компьютере откройте файл `.env` в корневой папке бота.
2. Измените параметры `BYPASS_HOST` и `BYPASS_PORT`:
   ```env
   BYPASS_HOST=IP_НОВОГО_СЕРВЕРА
   BYPASS_PORT=443
   ```
3. Запустите скрипт деплоя для отправки изменений на основной сервер:
   ```bash
   node scratch/deploy.js
   ```

После этого бот автоматически перезапустится, и все новые генерируемые ссылки для обхода блокировок будут использовать новый IP-адрес и стандартный порт `443` (или `80`), проходя через ваш новый белый транзитный VPS.
